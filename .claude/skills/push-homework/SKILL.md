# 推送作业 (push-homework)

将本地更改完整地推送到 GitHub 远程仓库。自动处理 Git 提交与推送的完整流程。

## 工作流程

1. **git add .** — 添加所有更改到暂存区
2. **git status** — 检查是否有变更需要提交
3. **git commit** — 生成带当前日期时间的提交信息，如 "作业提交 2026-04-15 14:30"
4. **git pull --rebase** — 先拉取远程最新代码（防止非快进拒绝）
5. **git push** — 推送到远程仓库

## 行为规则

- 无变更时直接退出，不创建空提交
- LF/CRLF 警告不影响执行，继续正常提交流程
- 如果 pull 后有合并，自动完成合并后再 push
- 提交信息格式：`作业提交 YYYY-MM-DD HH:MM:SS`

## 使用方式

当用户说「推送作业」时，直接执行完整流程。执行完成后报告结果。
